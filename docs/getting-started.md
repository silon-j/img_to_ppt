# 新手指引

## 方式一：直接下载安装（推荐给非技术用户）

1. 前往 [GitHub Releases](../../releases) 页面
2. 下载最新版本的 `.dmg` 文件：
   - Apple Silicon Mac（M 系列）→ `图片转PPT-arm64.dmg`
   - Intel Mac → `图片转PPT-x64.dmg`
3. 打开 `.dmg`，将 **图片转PPT** 拖入「应用程序」文件夹
4. 双击启动

> **macOS 安全提示**：首次打开时系统可能提示"无法验证开发者"，在「系统设置 → 隐私与安全性」中点击「仍要打开」即可。

## 方式二：本地开发运行

### 环境要求

- [Node.js](https://nodejs.org/) >= 18（建议通过 [nvm](https://github.com/nvm-sh/nvm) 管理）
- npm >= 9（随 Node.js 一起安装）

### 步骤

```bash
# 1. 克隆仓库
git clone https://github.com/你的用户名/img_to_ppt.git
cd img_to_ppt

# 2. 安装依赖（在 app 目录下）
cd app
npm install

# 3. 启动应用
npx electron .
```

## 使用方法

启动后界面分三个区域：

### 1. 选择图片文件夹

点击「**选择**」按钮，在弹出的文件夹选择器中找到存放图片的目录。

支持的图片格式：PNG、JPG/JPEG、BMP、GIF、TIFF

> 图片会按**文件名自然排序**（数字按大小、中文按拼音），与 Finder 排列顺序一致。

### 2. 选择保存位置

点击「**选择**」按钮指定 `.pptx` 文件的保存路径和文件名。

> 选完图片文件夹后，保存路径会自动填入（同级目录 `output.pptx`），通常无需手动修改。

### 3. 自适应压缩图片（可选）

勾选此选项后，生成前会自动对图片进行压缩：

- 超过 2560×1440 分辨率的图片会等比缩小
- 不含透明通道的图片转为 JPEG（质量 85），体积通常减少 60-90%
- 含透明通道的 PNG 使用高压缩级别编码

> 对于 PPT 播放场景，压缩后的画质与原图**肉眼无法区分**，但文件体积大幅减小。如需保留原始画质（如存档用途），可取消此勾选。

### 4. 生成 PPT

点击「**生成 PPT**」按钮，进度条会实时显示当前处理的图片。

生成完成后弹窗显示：
- 总页数
- 保存路径
- 如果开启压缩：图片原始总大小 → 压缩后大小（节省百分比）

## 常见问题

**Q：生成的 PPT 可以在 Windows 上打开吗？**
A：可以，输出的 `.pptx` 是标准格式，兼容 PowerPoint、WPS 等。

**Q：图片顺序不对怎么办？**
A：工具按文件名排序，建议在文件名中加入序号（如 `01_封面.png`、`02_内容.png`）确保顺序正确。

**Q：支持子文件夹吗？**
A：目前只扫描所选文件夹的根目录，不递归子文件夹。

**Q：PPT 的幻灯片尺寸是多少？**
A：固定为 16:9 宽屏（13.333 × 7.5 英寸），与主流 PPT 模板一致。
